# Especificações do projeto

## Histórias de usuário
- Como usuário, quero me cadastrar e autenticar para acessar ferramentas protegidas.
- Como usuário, quero executar ferramentas de rede e gerar relatórios em PDF.

## Critérios de aceitação
- O cadastro deve criar um usuário com senha criptografada.
- O login deve retornar um token JWT válido.
- O backend deve responder com erros claros para entradas inválidas.
